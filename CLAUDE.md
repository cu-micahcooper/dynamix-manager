# CLAUDE.md

## Project Overview

CIO-level executive analytics workspace for Cedarville University's TeamDynamix environment. Ingests survey and ticket data from the TeamDynamix API, stores it in a local DuckDB database, and generates HTML reports and Jupyter notebooks.

## Environment Setup

```bash
./scripts/bootstrap_env.sh   # creates .venv, installs package + dev deps
source .venv/bin/activate
```

Requires Python 3.13+. Credentials in `.env` (never commit):

```
TDX_BASE_URL=https://your-instance.teamdynamix.com
TDX_APP_ID=123
TDX_USERNAME=svc-account@example.edu
TDX_PASSWORD=...
```

## Verification Commands

Run after any change:

```bash
python -m pytest
python -m ruff check src/ tests/
```

All tests must pass and ruff must be clean before considering work complete.

## CLI Entry Point

```bash
dynamix-manager refresh-surveys
dynamix-manager backfill-tickets [--batch-size N] [--max-batches N]
dynamix-manager quality-check <ticket_app_id> [--limit N]
dynamix-manager cache-days-off
dynamix-manager generate-report
dynamix-manager generate-email
```

## Project Structure

```
src/dynamix_manager/   # main package
  cli.py               # CLI entry point (argparse)
  config.py            # env var loading
  tdx_client.py        # TeamDynamix API client
  storage.py           # DuckDB read/write
  models.py            # dataclasses / schema
  pipeline.py          # orchestration
  surveys.py           # survey ingestion
  tickets.py           # ticket ingestion
  metrics.py           # survey metrics
  ticket_quality.py    # open-ticket quality flags
  executive.py         # executive report data
  reporting.py         # HTML report generation
  notebooks.py         # notebook generation
  planned_days_off.py  # days-off caching
tests/                 # pytest; mirrors src module names
scripts/               # one-off data scripts (not part of package)
notebooks/             # Jupyter notebooks (generated by CLI)
reports/               # HTML reports (generated by CLI)
data/analytics.duckdb  # local DuckDB file (not checked in)
```

## Key Conventions

- **Python 3.13+**, no type annotations required unless already present in a file
- **ruff** for linting — keep clean at all times
- **pytest** — keep all tests passing; add tests for new behavior
- **DuckDB** for all persistence — no SQLite, no ORM
- HTML reports are self-contained (inline JS/CSS); use `esc()` helper for XSS safety
- Plotly charts embedded via `window.*_DATA` JSON blobs — escape `</` to prevent script injection
- Do not commit `.env`, `data/`, `reports/`, or `.venv`

## Workflow Preferences

- Run `python -m pytest && python -m ruff check src/ tests/` before claiming any task is done
- Prefer editing existing files over creating new ones
- Keep solutions minimal — no unnecessary abstractions or future-proofing
